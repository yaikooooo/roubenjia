# 图片格式支持功能说明

## 功能概述

已成功实现对PNG和GIF两种格式的全面支持，包括：

1. **LOGO和Banner支持PNG/GIF双格式**
2. **奖品图片分场景使用不同格式**
   - 抽奖页面：使用静态PNG图片
   - 结算界面：使用动态GIF图片

## 目录结构

```
assets/
├── LOGO1.png         # LOGO的PNG版本
├── LOGO.gif          # LOGO的GIF版本（可选）
├── BABNNER.png       # Banner的PNG版本
├── BABNNER.gif       # Banner的GIF版本（可选）
├── reward_item/      # 抽奖页面使用的静态PNG
│   ├── P01.png
│   ├── P02.png
│   └── ... (P01-P12)
└── showward/         # 结算界面使用的动态GIF
    ├── P01.gif
    ├── P02.gif
    └── ... (P01-P12)
```

## 实现原理

### 1. LOGO和Banner自动格式检测

- 系统启动时自动检测GIF版本是否存在
- 如果存在GIF版本，优先使用GIF
- 如果不存在GIF版本，回退到PNG版本
- 检测结果保存在组件的data中：
  - `logoHasGif`: LOGO是否有GIF版本
  - `bannerHasGif`: Banner是否有GIF版本

### 2. 奖品图片分场景加载

- **抽奖页面** (`getPrizeMediaUrl`方法)：
  - 从 `assets/reward_item/` 目录加载PNG格式
  - 文件命名：P01.png, P02.png, ..., P12.png

- **结算界面** (`getModalMediaUrl`方法)：
  - 从 `assets/showward/` 目录加载GIF格式
  - 文件命名：P01.gif, P02.gif, ..., P12.gif

## 核心方法

### 图片URL获取方法

```javascript
// LOGO图片URL
getLogoImageUrl() {
  const hasGif = this.logoHasGif !== undefined ? this.logoHasGif : false;
  return hasGif ? 'assets/LOGO.gif' : 'assets/LOGO1.png';
}

// Banner图片URL
getBannerImageUrl() {
  const hasGif = this.bannerHasGif !== undefined ? this.bannerHasGif : false;
  return hasGif ? 'assets/BABNNER.gif' : 'assets/BABNNER.png';
}

// 抽奖页面奖品图片URL（静态PNG）
getPrizeMediaUrl(index) {
  const fileName = 'P' + (index + 1).toString().padStart(2, '0') + '.png';
  return `assets/reward_item/${fileName}`;
}
```

### 异步格式检测

```javascript
// 检测图片格式
async detectImageFormats() {
  this.logoHasGif = await this.checkImageExists('assets/LOGO.gif');
  this.bannerHasGif = await this.checkImageExists('assets/BABNNER.gif');
}

// 检查图片是否存在
checkImageExists(url) {
  return new Promise((resolve) => {
    const img = new Image();
    img.onload = () => resolve(true);
    img.onerror = () => resolve(false);
    img.src = url;
    setTimeout(() => resolve(false), 3000); // 3秒超时
  });
}
```

## 使用方法

### 1. 基础使用（仅PNG）

只需要准备PNG格式的文件：
- `assets/LOGO1.png`
- `assets/BABNNER.png`
- `assets/reward_item/P01.png` ~ `P12.png`
- `assets/showward/P01.gif` ~ `P12.gif`

### 2. 高级使用（PNG+GIF）

如果需要动态效果，可以额外提供GIF版本：
- `assets/LOGO.gif` （可选）
- `assets/BABNNER.gif` （可选）

系统会自动检测并优先使用GIF版本。

## 注意事项

1. **文件命名规范**：
   - 奖品图片必须按照 P01, P02, ..., P12 的格式命名
   - 使用两位数字，不足补零

2. **目录结构**：
   - `reward_item` 目录存放抽奖页面的静态PNG
   - `showward` 目录存放结算界面的动态GIF
   - 两个目录中的文件名必须一致

3. **性能考虑**：
   - GIF文件通常比PNG文件大，注意文件大小
   - 系统启动时会异步检测GIF文件，不会阻塞页面加载

4. **兼容性**：
   - 如果GIF文件不存在，系统会自动回退到PNG版本
   - 确保PNG版本始终存在作为备选方案

## 扩展说明

如果需要手动控制使用哪种格式，可以直接修改组件data中的标志：

```javascript
data() {
  return {
    logoHasGif: true,    // 强制使用LOGO的GIF版本
    bannerHasGif: false, // 强制使用Banner的PNG版本
    // ...
  }
}
```

这样可以跳过自动检测，直接使用指定格式。
