# 抽奖系统

一个纯本地运行的桌面抽奖系统，默认通过 Electron 启动，不依赖任何自建后端。

## 功能特点

- 登录页（管理员权限）
- 奖池配置（支持多组奖池）
- 转盘抽奖动画
- 根据奖品权重随机抽奖
- 抽奖历史记录
- 本地桌面运行，无需部署服务器

## 技术栈

- Vue 3 (CDN版)
- Vue Router
- Tailwind CSS

## 快速开始

1. 克隆本仓库
```bash
git clone <repo-url>
```

2. 本地运行

### 桌面模式

- 直接双击 `start-desktop.bat` 启动桌面版窗口（推荐）
- 或在命令行执行：`npm run desktop`
- 无边框窗口模式可执行：`npm run desktop:frameless`

### 网页模式

- 直接双击 `start-web.bat` 启动本地网页服务并自动打开浏览器
- 网页模式会先构建 `vendor/tailwind.css`，再生成 `assets/sounds/manifest.json`
- 默认地址为 `http://127.0.0.1:9527/index.html`，若端口被占用会自动顺延下一个可用端口

## 桌面版说明

### 奖池兼容策略

- 应用启动时会优先读取项目内的 `reward-config.json`
- 若内置配置读取失败，自动回退到本地快照 `localStorage.rewardConfigSnapshot`
- 管理员在配置页保存后，会把最新配置写入本地 `localStorage.rewardConfigLocal`

## 使用说明

1. 首次使用请登录为管理员（用户名: admin / 密码: 123456）
2. 进入配置页面创建奖池，设置奖品信息和图片
3. 在选择奖池页面选择要使用的奖池
4. 返回首页，点击"开始抽奖"进行抽奖

## 组件说明

- 抽奖页正式实现文件为 `components/Draw.js`
- `Draw1.js` 已废弃并移除，不再参与当前桌面版构建
- 当前奖池布局已从 `12` 个奖励位升级为 `16` 个奖励位，采用四边矩形环绕布局，方便继续微调上下左右数量与间距
- `reward-config.json` 中的 `ui_config.layout` 现在控制 `16` 奖位的横排/竖排数量、边距和中心位置，后续微调优先改这里
- 若需要精确调整上下左右到中间的距离，以及左右两列的上下高度，可直接修改 `ui_config.layout.topRowY`、`bottomRowY`、`leftColumnX`、`rightColumnX`、`sideColumnTopY`、`sideColumnBottomY`
- 抽奖页布局引擎已改为“配置加载完成后再计算坐标”，避免页面先用默认值渲染、后续配置生效却不重算的问题
- 抽奖舞台坐标系固定按 `1920x1080` 设计尺寸生成，保证已拼好的 UI 坐标不变；“界面设置”里的尺寸预设只调整输出窗口与最终等比例缩放
- 抽奖页动效已规划为三层：奖项跳动命中粒子爆点、常规中奖微动效、大奖全屏特效，统一走本地动画库和本地粒子实现，避免 CDN 依赖
- 命中粒子当前使用星芒形小粒子，不再使用圆点，便于和奖品边框的爆闪效果统一
- 中间 `LOGO` 图像本体上方使用十字星芒闪光，星芒坐标固定基于设计稿内的 `LOGO` 区域计算，再随舞台整体等比缩放，不会因为页面尺寸变化而重新漂移；星芒缩放动画已调整为连续放大后顺滑回落，避免中段停顿；流星拆成页面顶层与次顶层两层，整体方向统一偏左上到右下，但会基于整屏随机锚点覆盖更多区域，不再只集中在单一对角线路线上，头部会做亮暗与大小变化
- 命中爆点、`LOGO` 星芒、`BANNER` 流星的时长随机区间已统一放大为当前视觉时长的 `1x ~ 2x`
- `BANNER` 流星的存在时长已额外整体翻倍，便于在大屏和电视场景下更容易观察到拖尾变化
- 结算奖励弹窗已改为和主舞台共用同一套等比缩放，不会再按窗口单独算一套尺寸；弹窗奖品图片入场动画也已去掉会造成左右翻转的 3D 旋转
- 结算弹窗的最终缩放现在在首帧即生效，弹窗只保留轻量入场动画，不再出现“先按设计尺寸全量渲染再缩回去”的闪动
- 奖项可通过 `effectTier: jackpot` 或 `isJackpot: true` 配置为大奖，全屏特效不会写死奖项名称
- `reward-config.json` 中的 `ui_config.ui_settings` 现在控制奖品尺寸、Banner 尺寸和 LOGO 比例，避免在业务代码里改死
- “界面设置”已支持保存 `ui_config.stage.preset`，当前仅提供 `默认` 与 `3240 拼接屏` 两档预设；切换后只会自动映射桌面开窗尺寸，舞台仍保持 `1920x1080` 坐标系并做等比例缩放，重启桌面应用可确保完全生效
- 桌面 `--frameless` 无边框模式下，页面顶部会保留独立拖拽条用于移动窗口，不影响业务区既有布局和等比例缩放
- 资源检测脚本已改为按 `reward-config.json` 中实际引用的资源路径检查，重复奖励复用旧资源时不再强制要求单独创建 `P13-P16` 文件
- 设置页的背景音乐默认使用 `随机播放 BJ 目录`，也支持切换为 `指定单曲循环` 或 `关闭背景音乐`
- 背景音乐清单不再在页面里写死，启动时会根据 `assets/sounds/BJ/` 实际文件自动生成 `assets/sounds/manifest.json`
- 抽奖页会按背景音乐模式实际播放：`random` 会在清单内连续随机切歌且尽量避免连续重复，`single` 只循环当前选中的那一首
- `assets/sounds/BJ/` 目录背景音乐如需优化循环衔接，可统一批量裁掉尾部 `1s`，裁剪后直接覆盖原文件，再重新生成音频清单即可
- 抽奖页新增 `ui_config.sounds.voiceover` 配置段，统一管理欢迎语、点击开始、即将停下、常规中奖、稀有中奖、头奖、长时间未操作等配音节点，业务代码不再写死配音文件名
- 配音资源目录固定为 `assets/sounds/peiyin/`，当前最新规则为：点击开始后立刻从 `start` 的 `28` 条文案中随机播放 `1` 条；转盘即将停下前从 `countdown` 的 `4` 条文案中随机播放 `1` 条；中奖后按奖项专属节点或奖项层级节点播放中奖配音
- 奖品语音优先读取奖项自身的 `voiceoverNode` 指向专属配音节点；未配置专属节点时再回退读取 `voiceoverTier`，可配置为 `regular`、`rare`、`jackpot`
- 长时间未操作配音改为由配置控制触发时长，当前默认使用 `30000ms`，进入抽奖页后自动开始计时，任意交互会重置空闲计时器
- `start`、`countdown`、中奖配音节点现已改为串行衔接：当前语音未播完时，后续节点会排队等待，不再直接截断前一句；进入更后阶段时会自动清掉排队中的旧阶段语音，避免转场后还继续播过期提示
- 普通抽奖跑马灯动画时长现统一固定为 `10s`，`countdown` 即将停下配音会比最终停下点提前约 `3.8s` 触发
- 生产资源使用本地构建的 Tailwind CSS 和 Vue 生产版脚本，避免浏览器中的 CDN/开发版告警影响正式环境
- 神秘奖励会继续保留 `type: mystery` / `isMystery: true` 状态逻辑，但奖格本身不再默认常驻黄色描边，避免与抽奖过程高亮混淆

## 数据存储

- 用户权限: localStorage.isAdmin
- 当前奖池: localStorage.lastGroupId
- 抽奖记录: localStorage.lotteryLog
- 奖池配置（默认）：localStorage.rewardConfigLocal（覆盖本地 reward-config.json）
- 奖品资源（默认）：本地 assets/
